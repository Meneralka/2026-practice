# 1. Типы документов (актуализированная матрица из Разделов 5 и 8)

| Тип документа (`documentType`) | Источник | Использование / Описание | Допустимые статусы (`status`) | Типовые переходы (ЖЦ) |
| :--- | :--- | :--- | :--- | :--- |
| **Материалы заявки** (`REQUEST_MATERIAL`) | Заявитель | Проверка комплектности, основание для проведения экспертизы. | Черновик, На проверке, Принят, Требуется замена, Архив. | Черновик $\rightarrow$ На проверке $\rightarrow$ Принят $\rightarrow$ Архив.<br>При замечаниях: На проверке $\rightarrow$ Требуется замена $\rightarrow$ На проверке. |
| **Экспертная карта** (`EXPERT_CARD`) | Эксперт[cite: 5] | Выводы, промежуточный результат (содержательная часть — в `expertise-service`). | Черновик, На проверке, Принят, Требуется замена, На согласовании, Утвержден, Архив. | Черновик $\rightarrow$ На проверке $\rightarrow$ Принят $\rightarrow$ На согласовании $\rightarrow$ Утвержден $\rightarrow$ Архив.<br>При замечаниях: На проверке/На согласовании $\rightarrow$ Требуется замена. |
| **Экспертное заключение** (`EXPERT_CONCLUSION`) | Система / эксперт[cite: 5] | Итоговый документ для заявителя.[cite: 5] | Черновик, На проверке, Требуется замена, На согласовании, Утвержден, Направлен заявителю, Архив. | Черновик $\rightarrow$ На проверке $\rightarrow$ На согласовании $\rightarrow$ Утвержден $\rightarrow$ Направлен заявителю $\rightarrow$ Архив.<br>При ошибке файла: На проверке $\rightarrow$ Требуется замена. |
| **Сравнительный протокол** (`COMPARATIVE_PROTOCOL`)[cite: 5, 6] | Проверка[cite: 5] | Результат повторной проверки/допэкспертизы со связью с первичной экспертизой.[cite: 5, 6] | Черновик, На проверке, Принят, Требуется замена, На согласовании, Утвержден, Архив. | Черновик $\rightarrow$ На проверке $\rightarrow$ Принят $\rightarrow$ На согласовании $\rightarrow$ Утвержден $\rightarrow$ Архив.<br>При замечаниях: На проверке/На согласовании $\rightarrow$ Требуется замена. |
| **Служебная резолюция** (`SERVICE_RESOLUTION`)[cite: 5, 6] | Внутренний пользователь[cite: 5] | Поручение, согласование, комментарий. Внутренний документ (не публикуется заявителю).[cite: 5, 6] | Черновик, На согласовании, Утвержден, Архив. | Черновик $\rightarrow$ На согласовании $\rightarrow$ Утвержден $\rightarrow$ Архив. |

# 2. Статусы документа и сервисы-инициаторы (Раздел 8.1)

| Статус | Что означает | Сервис-инициатор перехода | Основание для перехода |
| :--- | :--- | :--- | :--- |
| **Черновик**[cite: 5, 6] | Файл загружен, но не включен в завершенное действие.[cite: 5] | `document-service` / сервис-источник | Создание карточки или загрузка новой версии файла. |
| **На проверке**[cite: 5, 6] | Идет техническая/антивирусная проверка или комплектностная проверка.[cite: 5] | `document-service`, `request-service`, `expertise-service`, `review-service` | Техническая проверка файла или проверка содержания владельцем бизнес-объекта.[cite: 6] |
| **Принят**[cite: 5, 6] | Документ включен в материалы дела.[cite: 5] | `request-service`, `expertise-service`, `review-service`, `document-service`[cite: 6] | Документ официально включен в материалы заявки/дела/проверки.[cite: 6] |
| **Требуется замена**[cite: 5, 6] | Отклонен с замечанием (требуется загрузка новой `FileVersion`).[cite: 5, 6] | `request-service`, `expertise-service`, `review-service`, `workflow-service`[cite: 6] | Формализованное замечание, требующее исправления.[cite: 6] |
| **На согласовании**[cite: 5, 6] | Проходит маршрут ЭДО.[cite: 5] | `workflow-service`, `report-service`, `document-service`[cite: 6] | Запущен маршрут ЭДО или процесс согласования итогового документа.[cite: 6] |
| **Утвержден**[cite: 5, 6] | Принят ответственным лицом.[cite: 5] | `workflow-service`[cite: 6] | Маршрут согласования завершен положительно.[cite: 6] |
| **Направлен заявителю**[cite: 5, 6] | Опубликован в личном кабинете.[cite: 5] | `report-service`, `cabinet-service`, `document-service`[cite: 6] | Документ опубликован по профилю видимости.[cite: 6] |
| **Архив**[cite: 5, 6] | Закрыт для изменений, доступен только на просмотр.[cite: 5] | `workflow-service`, `document-service`, `report-service`[cite: 6] | Документ закрыт для изменений и отправлен на регламентное хранение.[cite: 6] |

# 3. Разделение бизнес-данных и метаданных

## 3.1. Зона ответственности `document-service`
* Владеет карточкой документа (`DocumentCard`), версиями файла (`FileVersion`), метаданными, связями документа с бизнес-объектами (`DocumentLink`), статусом в части документооборота и журналом действий (аудит).[cite: 5, 6]
* Выдает защищенные временные ссылки (`secureLinkRef`) по запросу.[cite: 6]

## 3.2. Реквизиты, принадлежащие внешним сервисам (Раздел 3.2 и 12.1)
Связи осуществляются исключительно через идентификаторы бизнес-объектов (`targetId`). Прямой доступ к базам данных смежных сервисов категорически запрещен.[cite: 5, 6]

* **`request-service`**: Владеет заявкой, рассчитывает комплектность, управляет требованиями к комплекту.[cite: 6]
* **`expertise-service`**: Владеет экспертным делом, критериями оценки, расчетами, методиками и текстовыми выводами эксперта.[cite: 5, 6]
* **`review-service`**: Владеет содержанием проверки и результатами сравнения в протоколе.[cite: 5, 6]
* **`report-service`**: Владеет текстом итогового вывода/решения и принимает решение о публикации заключения.[cite: 5, 6]
* **`cabinet-service`**: Отображает данные заявителю, фиксирует факты открытия и скачивания (формирует `DeliveryReceipt`).[cite: 6]
* **`workflow-service`**: Владеет маршрутом, задачами ЭДО, SLA и решениями согласующих лиц.[cite: 6]
