# MZD-57: Компонентная структура пакетов document и delivery

Архитектурный реестр классов и модулей, определяющий строгие зоны ответственности в соответствии с ФС-1.

## 1. Пакет `document/` (Управление документами)

* **DocumentController** — REST-API для работы с метаданными карточек документов и инициации процессов.
* **FileController** — REST-API для загрузки, скачивания и версионирования физических файлов документов.
* **DocumentService** — Оркестратор бизнес-логики, отвечающий за валидацию, создание и изменение метаданных.
* **DocumentValidationPipeline** — Последовательный пайплайн технической и антивирусной проверки загруженного файла: формат (Tika), размер, целостность (SHA-256), антивирус (ClamAV) (MZD-61 §1).
* **FileStorageService** — Сервис интеграции с объектным хранилищем (S3) и первичной регистрации версий файлов, включая перенос из карантина в основное хранилище и генерацию защищенных временных ссылок скачивания/предпросмотра (MZD-61 §2).
* **DocumentCard** — JPA-сущность, представляющая собой агрегат метаданных карточки документа.
* **FileVersion** — JPA-сущность, содержащая метаданные физического файла, его хэш-сумму и статус антивирусного сканирования.
* **DocumentRepository** — Интерфейс доступа к базе данных для манипуляций с сущностями `DocumentCard`.
* **FileVersionRepository** — Интерфейс доступа к базе данных для работы с версиями файлов `FileVersion`.
* **DocumentStateMachine** — Компонент-валидатор переходов состояний документов на основе тип-специфичных графов (MZD-22).

## 2. Пакет `delivery/` (Управление доставкой в ЛК)

* **DeliveryController** — REST-API для мониторинга статусов доставки документов внешним потребителям.
* **DeliveryService** — Бизнес-сервис, управляющий процессами отправки документов в ЛК и обработкой квитанций.
* **DocumentPublishingService** — Публикует проверенный документ в ЛК заявителя: проверяет статус `READY` и тип документа, вычисляет `visibilityProfile`, переводит `DocumentCard` в `PUBLISHED`, регистрирует событие `DocumentPublished` в Outbox (MZD-61 §3).
* **DeliveryReceiptService** — Регистрирует юридически значимые квитанции `DELIVERED`/`OPENED`/`DOWNLOADED` по композитному ключу `[documentId, recipientId, receiptType]`, работает по WORM-модели (MZD-61 §4).
* **DeliveryReceipt** — JPA-сущность, фиксирующая факт, время и статус получения/публикации документа в ЛК.
* **DeliveryRepository** — Интерфейс доступа к базе данных для хранения и поиска квитанций доставки `DeliveryReceipt`.
