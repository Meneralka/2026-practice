# Каталог тест-кейсов · document-service

**Принцип:** на каждый сценарий — позитивный и негативный кейс. Покрыты все исключения, не только успешный путь.
**Источники:** реестр REQ-08 (ТЗ р.8), SPEC-04 (@koshak1030), SPEC-09 (@ArtemKvvs), MZD-22 (стейт-машины), MZD-46 (безопасность), SPEC-14 (envelope), ТЗ р.10, ФТ-05/07/29, НФТ.

---

## Легенда

| Обозначение | Значение |
|---|---|
| **SCC** | Spring Cloud Contract (граница: HTTP-стаб / Kafka-message) |
| **INT** | Интеграционный / `@SpringBootTest` (внутреннее состояние) |
| **MUST / SHOULD** | Приоритет требования (MUST блокирует релиз) |

**Справочник кодов (один кейс — один код):**

| Причина | Код |
|---|---|
| Неразрешённый MIME; тело файла не на upload-endpoint | 415 |
| Мутация версии; операция над карантином; конфликт `idempotencyKey`; операция над документом не в нужном статусе | 409 |
| Истёкшая ссылка | 410 |
| Несуществующий объект | 404 |
| Нарушение бизнес-правила / матрицы переходов | 422 |
| Отказ доступа; guard `SERVICE_RESOLUTION` | 403 |
| Нет/невалидный JWT | 401 |
| Недоступность смежного сервиса | 503 |

**Статусы документа:** `DRAFT`, `UNDER_REVIEW`, `ACCEPTED`, `REPLACEMENT_REQUIRED`, `UNDER_APPROVAL`, `APPROVED`, `PUBLISHED`, `ARCHIVED`
**Статусы файла (ФТ-07):** `SCANNING`, `CLEAN`, `INFECTED`
