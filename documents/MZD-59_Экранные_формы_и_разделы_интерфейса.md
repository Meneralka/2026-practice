# **Инвентарь экранов модуля загрузки документов и ЭДО (TZ-08)**

Документ описывает состав полей и доступные действия для каждого ключевого экрана системы в соответствии с разделом 9 ТЗ. Поля базируются на проекте OpenAPI-контрактов (DTO). Элементы, ожидающие финального утверждения в спецификации бэкенда, отмечены маркером *(предварительно)*.

## **1\. Экран загрузки файлов (Upload View)**

Экран для инициации процесса документооборота и прикрепления файлов к сущностям.  
**Состав полей (Форма загрузки):**

* files *(предварительно)* — массив бинарных файлов (Multipart/form-data).  
* documentType *(предварительно)* — выпадающий список типов (Материалы заявки, Экспертная карта и т.д.). *Обязательное поле.*  
* businessKey *(предварительно)* — ID связанной сущности (заявки, дела). *Обязательное поле.*  
* Индикаторы загрузки файла — progress (0-100%), uploadStatus (Успех/Ошибка), errorMessage (текст ошибки клиентской/серверной валидации).

**Доступные действия:**

* **Множественный выбор файлов:** через Drag & Drop или системный диалог.  
* **Черновое сохранение метаданных:** автосохранение введенного businessKey и documentType.  
* **Удаление/Очистка:** убрать файл с ошибкой из очереди загрузки.  
* **Отправка:** инициировать отправку валидных файлов на сервер (с поддержкой частичного успеха).

## **2\. Список документов с фильтрами (List View)**

Реестр или очередь обработки с поддержкой массивов данных, пагинации и поиска.  
**Состав полей (Отображение в таблице из DTO):**

* id — внутренний идентификатор документа.  
* businessKey — номер связанной заявки/дела.  
* title — название/имя актуального файла.  
* status — бейдж текущего статуса (На проверке, Утвержден и т.д.).  
* version — номер текущей версии (например, 1.2).  
* assignee.name и assignee.role — ФИО и роль ответственного.  
* deadline — крайний срок этапа.  
* updatedAt — дата последнего изменения.

**Состав полей (Фильтры, *предварительно*):**

* searchQuery — полнотекстовый поиск.  
* statusFilter — множественный выбор статусов.  
* typeFilter — фильтр по типу документа.  
* dateRange — период изменения/создания.

**Доступные действия:**

* **Навигация:** сортировка по столбцам, пагинация (pageNumber, pageSize).  
* **Массовые действия:** выбор нескольких строк чекбоксами для массовой выгрузки архивом (если позволяют права).  
* **Переход:** клик по строке для открытия карточки документа.  
* **Сохранение представления:** *(предварительно)* сохранение набора фильтров как дефолтного.

## **3\. Карточка документа (Card View)**

Детальная информация о документе, его метаданных и истории версий без перезаписи.  
**Состав полей (Из DTO):**

* id, businessKey, type.  
* status — текущий статус с визуальной подсветкой.  
* nextRequiredAction — контекстная подсказка для пользователя (например, "SIGN\_AND\_SUBMIT").  
* metadata.authorId, metadata.createdAt — информация о создателе.  
* **Массив versions (История):**  
  * versionNumber (1.0, 1.1).  
  * fileName, fileSize.  
  * uploadedBy, uploadedAt.  
  * antivirusStatus — статус технической проверки.

**Доступные действия:**

* **Просмотр истории:** разворачивание аккордеона со старыми версиями файлов.  
* **Скачивание:** загрузка любой конкретной версии файла.  
* **Добавление версии:** кнопка загрузки новой ревизии файла (если статус позволяет).  
* **Редактирование реквизитов:** изменение типа или бизнес-ключа (с проверкой прав, формы поддерживают клиентскую валидацию).

## **4\. Экран маршрута (Route View)**

Визуализация маршрута ЭДО, согласования и утверждения.  
**Состав полей (Из DTO):**

* routeId, documentId, status (глобальный статус маршрута).  
* **Массив stages (Этапы):**  
  * stageOrder — порядковый номер (визуализация таймлайна).  
  * stageName — название этапа (Проверка комплектности, Экспертная оценка).  
  * status — статус этапа (COMPLETED, ACTIVE, PENDING).  
  * assignee.name — ответственный сотрудник.  
  * completedAt или deadline — фактическое время завершения или крайний срок.  
  * resolution — резолюция (Одобрено, Отклонено).

**Доступные действия (в зависимости от роли Руководитель/Координатор):**

* **Принятие решения:** кнопки "Утвердить", "Отклонить", "На доработку" (требует ввода комментария).  
* **Делегирование:** смена assignee для активного этапа.  
* **Напоминание:** генерация уведомления исполнителю, просрочившему deadline.

## **5\. Экран предпросмотра (Preview View / Split View)**

Разделенный экран для просмотра файла в браузере и работы с ним (например, заполнение экспертной карты).  
**Состав полей (Из DTO и ТЗ):**

* securePreviewUrl — временная ссылка для iframe/pdf-viewer.  
* mimeType — формат для выбора рендерера браузером.  
* previewAvailable — флаг успешной генерации превью.  
* **Поля правой панели (например, Экспертная карта, *предварительно*):**  
  * Блок динамических полей формы в зависимости от businessKey.  
  * draftLastSaved — метка времени последнего автосохранения.

**Доступные действия:**

* **Взаимодействие с файлом:** масштабирование, пагинация, поиск по тексту документа.  
* **Заполнение формы (Сплит-режим):** безопасный ввод данных с обязательным *черновым режимом* (автосохранение).  
* **Валидация:** подсветка незаполненных обязательных полей при попытке отправки результатов.  
* **Переход к скачиванию:** генерация secureDownloadUrl, если предпросмотр недоступен.

## **6\. Кабинет заявителя (Applicant Dashboard)**

Упрощенное рабочее пространство внешнего пользователя для отслеживания статусов и квитирования.  
**Состав полей (Композиция из List и Preview DTO):**

* **Дашборд *(предварительно)*:** счетчики заявок (inProgress, completed, requiresAction).  
* **Список:** title, status, updatedAt (без отображения внутренних резолюций и служебных полей).  
* **Квиток доставки (deliveryReceipt из DTO):**  
  * isDelivered (доставлено в кабинет).  
  * isOpen (документ просмотрен в браузере).  
  * isDownloaded (документ скачан локально).

**Доступные действия:**

* **Просмотр статуса:** навигация по списку своих заявок.  
* **Чтение уведомлений:** просмотр публичных замечаний (ошибки комплектности) без внутренних деталей (скрыт stack trace и служебная переписка).  
* **Квитирование (Фоновое действие):** при клике на просмотр или скачивание итогового заключения, UI отправляет запрос на обновление статусов в deliveryReceipt.  
* **Обновление ссылки:** запрос нового токена при истечении urlExpiresAt.
